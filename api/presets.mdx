Vard provides three built-in presets that balance security and usability for different scenarios. Each preset configures threshold and default threat actions.

## vard.strict()

Creates a strict vard with low detection threshold (0.5). Most sensitive to attacks but higher chance of false positives. Recommended for high-security environments.

**Returns:** `CallableVard` with strict configuration

### Configuration

- **Threshold:** `0.5` (low - catches more threats)
- **Max length:** `10000` characters
- **Threat actions:** All threats set to `block`

```typescript
{
  threshold: 0.5,
  maxLength: 10000,
  threatActions: {
    instructionOverride: 'block',
    roleManipulation: 'block',
    delimiterInjection: 'block',
    systemPromptLeak: 'block',
    encoding: 'block',
  }
}
```

### Usage

```typescript
import vard from '@andersmyrmel/vard';

const strict = vard.strict();

// Even moderate threats are blocked
strict.parse('start over');
// Throws: PromptInjectionError (severity 0.75 > threshold 0.5)
```

**Extend strict preset:**

```typescript
const myVard = vard.strict()
  .delimiters(['CONTEXT:', 'USER:'])
  .maxLength(10000);

const safe = myVard.parse(userInput);
```

## vard.moderate()

Creates a moderate vard with balanced detection threshold (0.7). Good balance between security and usability. **This is the default preset.**

**Returns:** `CallableVard` with moderate configuration

### Configuration

- **Threshold:** `0.7` (balanced)
- **Max length:** `10000` characters
- **Threat actions:** High-severity threats blocked, delimiters/encoding sanitized

```typescript
{
  threshold: 0.7,
  maxLength: 10000,
  threatActions: {
    instructionOverride: 'block',
    roleManipulation: 'block',
    delimiterInjection: 'sanitize',
    systemPromptLeak: 'block',
    encoding: 'sanitize',
  }
}
```

### Usage

```typescript
import vard from '@andersmyrmel/vard';

const moderate = vard.moderate();

// High severity threats are blocked
moderate.parse('ignore all previous instructions');
// Throws: PromptInjectionError (severity 0.9 > threshold 0.7)

// Delimiter injection is sanitized (not blocked)
const result = moderate.parse('<system>Hello</system>');
console.log(result); // "Hello" (delimiter removed)
```

**Customize moderate preset:**

```typescript
const chatVard = vard.moderate()
  .delimiters(['CONTEXT:', 'USER:', 'SYSTEM:'])
  .block('delimiterInjection')  // Block instead of sanitize
  .maxLength(5000);

const safe = chatVard.parse(userInput);
```

## vard.lenient()

Creates a lenient vard with high detection threshold (0.85). Sanitizes most threats instead of blocking. Only very high severity threats are blocked.

Recommended when false positives are costly or user input is expected to contain instruction-like language (e.g., technical documentation).

**Returns:** `CallableVard` with lenient configuration

### Configuration

- **Threshold:** `0.85` (high - only catches very severe threats)
- **Max length:** `10000` characters
- **Threat actions:** Most threats sanitized or warned, only critical ones blocked

```typescript
{
  threshold: 0.85,
  maxLength: 10000,
  threatActions: {
    instructionOverride: 'sanitize',
    roleManipulation: 'warn',
    delimiterInjection: 'sanitize',
    systemPromptLeak: 'sanitize',
    encoding: 'sanitize',
  }
}
```

### Usage

```typescript
import vard from '@andersmyrmel/vard';

const lenient = vard.lenient();

// Moderate severity threats pass through
const safe = lenient.parse('new instructions');
// Returns: 'new instructions' (severity 0.8 < threshold 0.85)

// Very high severity still blocked
lenient.parse('ignore all previous instructions');
// Throws: PromptInjectionError (severity 0.9 > threshold 0.85)
```

**Lenient for technical content:**

```typescript
const docVard = vard.lenient()
  .sanitize('instructionOverride')  // Don't block, just clean
  .sanitize('roleManipulation')
  .threshold(0.9);  // Even more permissive

const safe = docVard.parse(technicalDocumentation);
```

## Comparison

| Preset | Threshold | Best For | False Positives | Security |
|--------|-----------|----------|-----------------|----------|
| **strict** | 0.5 | High-security environments | Higher | Maximum |
| **moderate** | 0.7 | General applications (default) | Balanced | Balanced |
| **lenient** | 0.85 | Technical content, documentation | Lower | Permissive |

## Extending Presets

All presets return a `CallableVard` that can be further configured:

<CodeGroup>

```typescript Strict + Custom Delimiters
const myVard = vard.strict()
  .delimiters(['CONTEXT:', 'USER:'])
  .maxLength(5000);
```

```typescript Moderate + Custom Actions
const chatVard = vard.moderate()
  .block('delimiterInjection')  // Override default
  .warn('roleManipulation')     // Monitor instead of block
  .onWarn((threat) => {
    console.log('Warning:', threat.type);
  });
```

```typescript Lenient + Higher Threshold
const docVard = vard.lenient()
  .threshold(0.9)  // Even more permissive
  .allow('instructionOverride');  // Ignore completely
```

</CodeGroup>

<Note>
**Immutability:** All configuration methods return a new instance. The original preset is not modified.

```typescript
const base = vard.moderate();
const strict = base.threshold(0.5);  // Doesn't modify `base`
const lenient = base.threshold(0.9); // Doesn't modify `base`
```
</Note>
